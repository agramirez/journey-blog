## About

This is a blog! ☺

Yes, I know, that's not very helpful...but many programmers provide that as their documentation.  Not here!

No, proper documentation must contain the "why", "for what purpose", and "for whom".  In that spirit, the "why" of this blog is to document my journey into the Software Foundations world.  For what purpose?  To provide a record of the continous improvement that we undertake.

Just like no man was born 6' tall weighing 200 pounds, so too no project is born fully formed and functional.  There are some times when they are born that way, but for such projects to succeed one must be both a software engineer, software architect, AND an expert in the subject matter for which the software is written.

Since by definition I am new to the world of formal theorem provers, I cannot start with a fully functional system structure and methodology.  Therefore, we will take the tried and proven approach of Agile and continous improvement.  To this end we will use the spirit of formal methodologies such as TDD, BDD, and DDD.

Part of the "Behavior" and "Domain" will be documentated in this blog to explain and explore the setup and configuration of my projects:

- [Logical Foundations](https://github.com/agramirez/logical-foundations)
- [The Kitchen Sink](https://github.com/agramirez/the-kitchen-sink)
- [Agentic](https://github.com/agramirez/agentic)
